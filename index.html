「WEBコンテンツ作成」の成果

本テーマの学習目的

HTML, CSS, JavaScript の基本を学び構造の記述方法や見た目の設定、インタラクティブな処理を学び、簡単なWebページの作成を行えるようになる。また、デバックのやり方や写真の追加、検索結果を表示するなどをした。

プロジェクト課題について

今回のproject課題は天気のwebページの作成を行いました。それぞれの土地の名前をクリックすることによって、天気や最高気温、最低気温、湿度、気圧、風速を検索結果に表示するようにプログラムを行いました。JavaScriptのほうではquerySelectorを利用してイベント処理を行った。また、検索結果をajax通信でとってきたものをresultの結果に当てはめた。色合いは天気っぽい水色にして、検索結果を目立つように黄色にした。

    • 工夫点、苦心点、強調点、感想

    今回のproject課題ではテキストボックスを設置するのではなく文字をクリックすることで結果を表示するようにしました。これによって検索の手間が省けるようにしました。また、データを取得し検索結果を表示することに苦労した。強調点は見やすいサイトを心がけて作りました。クリックすることによる検索やシンプルな画面表示をするようにしました。感想:Webページ制作の授業で今まで開いていたサイトがいかに複雑で手の込んだものだと知った。第一回では、ｗｗｗとＨＴＭＬの基本を知ることができた。リストの作成の時にulやol、liなどの違いを知ることができた。第二回ではＣＳＳによる見た目の変更を学びました。特定の要素だけを修飾するという部分が少し複雑で難しかった。また、見た目の変更は、見やすい色合いにしたり見出しの色などの変更などを行ったが、公式のサイトのような色使いはまねできなかった。文字の大きさや文字体などは見やすい大きさに変更した。第三回ではJavaScriptの基本を学びました。Web開発ツールを使ってconsoleなどに表示するなどはJavaとは違う構文を使ったりしたが比較的わかりやすかった。Web開発者ツールのデバッガ機能は間違っている個所を教えてくれるので便利だと感じた。第四回のDOMとJavaScriptの授業は、とても難しく理解が大変だと感じた。プレースホルダ―としてのdiv要素の利用は少し複雑だった。document.querySelectorを使って要素を取得するなどのところや、CSSで写真を見やすくするなどのやり方が大変だった。第五回ではページ上でのイベント処理を学んだ。ボタンを押したときの処理は理解が大変だった。第六回ではproject課題をやった。検索の結果などを表示させるのはとても難しかった。今後Webページを作成するときはもっと色使いや配置などにも気を配りサイトを見た人が一目で分かるようなサイトを作れるように心がけたいです。また、clickしたら色が付くなども分かりやすいサイトを作るためには必要だと感じた。また、Web開発者ツールはとても便利なので、改めて学びなおしたいと思いました。文字を表示させるだけでも要素で囲んだり、script src の後のdeferは忘れがちだと思うのでしっかりと覚えておきたいです。フォントのサイズの替え方などもcssで作成し、背景などは様々な色の種類があり、グラデーションなどもすることができて幅広いと思いました。また、文字をゴシック体にしたりイタリック体にするやり方も学んだ。これらも用途に合わせて活用できたらいいなと思いました。普段見るサイトや拓殖大学のサイトはこの授業を学んだあと見てみるとclickするとURLに飛んだり写真を複数枚スライドで表示させたりと知らないことがたくさんあると感じた。
